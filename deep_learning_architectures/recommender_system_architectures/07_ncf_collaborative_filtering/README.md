# Neural Collaborative Filtering for Implicit Recommendation

This project implements an educational PyTorch version of Neural Collaborative
Filtering, NCF, using the NeuMF architecture on MovieLens 10M.

NCF is historically important because it replaces the fixed inner product used
in classical matrix factorization with learnable neural interaction functions.
Classical matrix factorization models a user-item score as:

```text
score(user, item) = user_embedding dot item_embedding
```

NCF generalizes this idea with neural networks. The original paper proposes GMF,
MLP, and NeuMF:

- GMF: generalized matrix factorization with an elementwise user-item product.
- MLP: nonlinear interaction modeling from concatenated user and item embeddings.
- NeuMF: a fusion of GMF and MLP signals.

This project is different from DIN and DIEN. DIN and DIEN model user behavior
sequences for CTR prediction. NCF models user-item collaborative filtering from
interaction data.

## Core Ideas

Collaborative filtering learns from user-item interaction patterns. Users with
similar interaction histories can imply similar item preferences, even when item
content features are not used.

Explicit feedback is direct preference data such as a 1 to 5 star rating.
Implicit feedback is behavioral data such as clicks, views, purchases, or
unobserved interactions. This project turns MovieLens explicit ratings into an
educational implicit-feedback binary classification problem.

User embeddings and item embeddings are trainable vectors looked up from encoded
integer ids. Matrix factorization compares those vectors directly. GMF keeps that
factorization idea by multiplying user and item embeddings elementwise. The MLP
branch concatenates user and item embeddings and learns nonlinear interaction
patterns. NeuMF concatenates the GMF vector and MLP output, then predicts one
interaction logit.

The model returns raw logits. Training uses `BCEWithLogitsLoss` because it
combines sigmoid and binary cross-entropy in a numerically stable operation.

## Label Construction

The preprocessing step builds binary interaction samples:

- Ratings greater than or equal to `positive_rating_threshold` become positive
  samples with `label = 1`.
- Negative samples are generated by sampling movies the user has not interacted
  with and assigning `label = 0`.

Each processed row contains:

```text
user_id,item_id,label
```

This is an educational approximation of implicit-feedback recommendation. In a
production recommender, negative sampling, temporal splitting, retrieval serving,
ranking calibration, and evaluation would need stricter design.

## Data Preparation

Run preprocessing once:

```bash
python scripts/prepare_data.py
```

The training script does not process raw MovieLens 10M. If the processed files
are missing, it prints a clear message asking you to run the preparation step.

## Training

```bash
python scripts/train.py
```

The validation set is used for model selection. The best checkpoint is selected
by validation AUC when AUC is available, otherwise by validation loss. The test
set is evaluated only once after training finishes.

## Prediction

```bash
python scripts/predict.py
```

The prediction script loads the saved checkpoint, reads a few rows from
`test.csv`, and prints:

- encoded user id
- encoded item id
- predicted interaction probability
- predicted class with threshold 0.5
- true label

Movie titles are optional metadata and are not required.

## Shapes

Inputs:

```text
user_ids: [batch_size]
item_ids: [batch_size]
```

Internal tensors:

```text
user embedding: [batch_size, embedding_dim]
item embedding: [batch_size, embedding_dim]
GMF vector: [batch_size, embedding_dim]
MLP input: [batch_size, 2 * embedding_dim]
fusion input: [batch_size, gmf_dim + mlp_last_dim]
```

Outputs:

```text
logits: [batch_size]
probabilities after sigmoid: [batch_size]
```

## Data Split

Samples are shuffled deterministically with `seed` and split into:

- training set
- validation set
- test set

With the default ratios, the split is 80% train, 10% validation, and 10% test.

## Memory Safety

MovieLens 10M is large, so this project defaults to `debug_mode=True`.

Important config options in `src/config.py`:

- `debug_mode`
- `max_users`
- `max_interactions`
- `max_samples`
- `batch_size`
- `num_workers`
- `n_epochs`

Full preprocessing can be enabled by editing `src/config.py`:

```python
debug_mode = False
max_users = None
max_interactions = None
max_samples = None
```

Full MovieLens 10M preprocessing and training may take time and memory. The
negative sampler uses rejection sampling and does not build a dense user-item
matrix or any `num_users * num_items` candidate structure.

## Model Summary and Key Papers

Neural Collaborative Filtering was introduced as a neural alternative to
classical matrix factorization for collaborative filtering. Its basic structure
uses embeddings for users and items, then learns interaction functions through
GMF, MLP, or their NeuMF fusion. It matters because it made user-item interaction
modeling more flexible than a fixed dot product while keeping the model
conceptually close to matrix factorization.

Key paper:

- Neural Collaborative Filtering
  Authors: Xiangnan He, Lizi Liao, Hanwang Zhang, Liqiang Nie, Xia Hu,
  Tat-Seng Chua
  Link: https://arxiv.org/abs/1708.05031
